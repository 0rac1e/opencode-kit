# OpenCode Kit

Reusable OpenCode configuration kit for any project. Provides a structured multi-agent development workflow with built-in planning, implementation, and code review phases.

## Language Profile

- Language: Go >= 1.24
- Commands: VERIFY=`go vet ./... && make fmt && make lint && make test`, FMT=`make fmt`, LINT=`make lint`, TEST=`make test`, VET=`go vet ./...`
- Source: `internal/**/*.go`, Generated: `*_gen.go`, Mocks: `*/mocks/*.go`
- Config: update `config.yaml.example` + `README.md` when config changes
- Concurrency: goroutines, channels, mutex, sync; race check: `go test -race`

> **Note:** Language Profile is project-specific. Override via `PROJECT-KNOWLEDGE.md` (generated by `/project-researcher`) or customize directly for your stack.

## Error Handling (All Agents)

| Error                                                                                  | Severity            | Action                                                                                         |
| -------------------------------------------------------------------------------------- | ------------------- | ---------------------------------------------------------------------------------------------- |
| Plan not found                                                                         | FATAL               | EXIT — run /planner first                                                                      |
| Plan not approved                                                                      | FATAL               | EXIT — run /plan-review first                                                                  |
| PROJECT-KNOWLEDGE.md missing                                                           | NON_CRITICAL        | Use profile above as defaults                                                                  |
| Review loop limit (3x)                                                                 | STOP_AND_WAIT       | Show iteration summary, request user help                                                      |
| Test/lint failure loop (3x)                                                            | STOP_AND_WAIT       | Load systematic-debugging skill → root cause investigation → if unresolved, request manual fix |
| Exploration budget exceeded (15 reads/context, 20 reads/sub-phase, ratio > 10 session) | STOP_AND_TRANSITION | Summarize findings, transition to next sub-phase                                               |
| Import violation                                                                       | STOP_AND_FIX        | Fix before proceeding                                                                          |
| Pre-commit build failure (`go build ./...`)                                            | STOP_AND_FIX        | Fix build errors before committing                                                             |

## Rules

- Architecture: `.opencode/rules/architecture.md` — import matrix, domain purity (active for `internal/**/*.go`)
- Go conventions: `.opencode/rules/go-conventions.md` — error wrapping, concurrency, config (active for `**/*.go`)
- Testing: `.opencode/rules/testing.md` — table-driven tests, race detector, mocks (active for `**/*_test.go`)
- Workflow: `.opencode/rules/workflow.md` — commands, agents, design rationale (global)

## Enforcement

- Permissions: configured in `opencode.json` — safe ops (make, go, git, Edit/Write) allowed, dangerous (rm -rf, git reset --hard, sudo) denied
- Agents: build (full access), plan (read-only), plan-reviewer (read-only), code-reviewer (read-only), code-researcher (read-only)
- Skills: on-demand loading via skill tool — workflow-protocols, planner-rules, coder-rules, code-review-rules
- Config: `opencode.json` (permissions + agents + commands)

## Conventions

- All artifacts use YAML-first format (>80% YAML, minimal prose)
- Language: English for code, YAML keys, and artifact specs
- Size limits enforced by agents (check-artifact-size)
- Examples use grep/glob patterns to find current code, not hardcoded snippets
